# 產品需求文件 (PRD)：股票追蹤 LINE Bot

## 1. 產品概覽與目標 (Product Overview)
*   **核心價值**：解決投資者需要頻繁切換 App 查看股價的痛點。透過 LINE 即可隨時查詢台股/美股行情，並管理個人自選股清單。
*   **目標受眾**：习惯使用 LINE 的個人投資者，需要輕量化、對話式的股票管理工具。

## 2. 功能規格與 AI 規格 (Specifications)
### 核心功能
1.  **即時股價查詢**：
    *   使用者輸入代號（如 `2330`）或名稱（如 `台積電`）。
    *   系統回傳：現價、漲跌幅、成交量。
2.  **自選股管理 (SQLite)**：
    *   新增自選：`新增 2330`。
    *   刪除自選：`刪除 2330`。
    *   查看清單：`我的清單`。
3.  **AI 自然語言處理**：
    *   使用 AI (Gemini 1.5 Flash) 解析使用者意圖。
    *   能識別「幫我查一下台積電」、「我要看 2330」等非固定指令。

### UI/UX 規格 (Premium Design)
*   **視覺表現**：股價查詢結果採用 **Flex Message**，包含漲跌顏色標註（漲紅跌綠）、股價走勢簡圖感（文字模擬或外部 API 圖片）。
*   **互動性**：清單顯示時附帶「刪除」按鈕，查詢結果附帶「加入自選」按鈕。

### AI 規格說明
*   **模型選擇**：Gemini 1.5 Flash。
*   **輸入 (Input)**：使用者傳送的 LINE 文字訊息。
*   **輸出 (Output)**：
    *   意圖解析結果 (JSON)：`{ "action": "query", "symbol": "2330" }`。
*   **延遲處理**：由於 LLM 生成需要時間，必須使用 **FastAPI BackgroundTasks** 異步處理訊息，確保 Webhook 立即回傳 200 OK。

## 3. 技術堆疊與性能指標 (Tech & Evaluation)
*   **技術堆疊**：
    *   後端框架：FastAPI。
    *   LINE SDK：line-bot-sdk-python v3。
    *   資料庫：SQLite (持久化存儲使用者清單)。
    *   股票資料源：`yfinance` (Yahoo Finance API)。
*   **評估指標 (KPIs)**：
    *   **延遲 (Latency)**：從收到訊息到回覆需在 3 秒內（符合 LINE Webhook 規範）。
    *   **精準度**：AI 辨識股票代號與操作意圖的準確率需達 95%。

## 4. 使用者體驗與異常處理 (UX & Fallback)
*   **互動路徑**：
    1. 使用者傳送訊息。
    2. AI 解析意圖並從資料庫/API 獲取資料。
    3. 系統回送回覆 (TextMessage 或 Flex Message)。
*   **降級處理 (Fallback)**：
    *   當 API 無法取得資料時，回傳：「抱歉，目前無法取得股價資訊，請稍後再試。」
    *   當 AI 無法辨識意圖時，回傳引導語：「您可以嘗試輸入『查詢 2330』或『新增 2330』。」

## 5. 安全與合規 (Safety & Compliance)
*   **投資建議聲明**：所有回覆訊息下方皆需標註「本工具僅供參考，不代表投資建議」。
*   **資料隱私**：僅存儲使用者的 LINE User ID 與自選股代號，不收集其他個資。
